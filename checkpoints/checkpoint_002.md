# BLF_WMS — Checkpoint 002 (Architecture & Navigation Design)

## 📅 Состояние проекта

Проект находится на этапе формирования полноценной архитектуры WMS системы.

Реализованы:
- Electron приложение
- ES Modules архитектура
- Shell (Header / Main / Footer)
- Map View (графическая карта склада)
- Table View (табличное представление остатков)
- базовая модель склада (Zone → Rack → Location)
- stock внутри локаций
- переключение Map ↔ Table

---

## 🧠 Принятая архитектурная модель

Система разделена на 4 ключевых слоя:

### 1. DATA LAYER (источник истины)
- warehouse.js
- структура склада
- остатки (stock)

### 2. UI LAYER (представления)
- Map View (карта)
- Table View (таблица)

### 3. SHELL LAYER (каркас приложения)
- Header (навигация)
- Main (контент)
- Footer (статус)

### 4. CONTROL LAYER (логика управления)
- view switching
- подготовка к movement system

---

## 🧭 Концепция системы

Принцип:

> Один источник данных → много представлений → действия через movements

---

## 📦 Принятая структура данных склада

Каждая локация:

- уникальный ID (A-01-01)
- zoneId / rackId / locId
- stock:
  - productId
  - series
  - qty

---

## 🧠 Проектирование навигации (вкладки системы)

Определена финальная концепция разделов приложения:

### 1. 🏬 СКЛАД (Warehouse)
Основной рабочий экран состояния склада:
- Map View
- Table View

---

### 2. 🔄 ОПЕРАЦИИ (Movements)
Центральный слой изменений:
- приход
- перемещение
- списание
- резервирование
- черновики операций

---

### 3. 📜 ИСТОРИЯ (History)
Аналитический слой:
- история по ячейке
- история по серии
- история по товару

---

### 4. 📘 ЛОГБУК (Audit Log)
Технический журнал системы:
- все движения (raw events)
- кто / когда / что сделал
- системные события

---

### 5. ⚙ АДМИН (будущее)
Административный слой:
- структура склада
- справочники
- настройки
- пользователи (позже)

---

## 🧠 Ключевые архитектурные решения

- склад НЕ изменяется напрямую
- все изменения идут через movements (будущий этап)
- history строится из movements
- logbook хранит полные события системы
- UI разделён на независимые views
